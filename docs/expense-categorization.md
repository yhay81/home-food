# 支出カテゴリ仕分けガイド（家計簿 expenses.haya.homes 用）

カワサキが受け取ったレシート・購入報告を、家計簿アプリ **expenses.haya.homes** へ **3層カテゴリ**で登録するための判断基準。
カテゴリ定義の実体は [`data/expense-categories.json`](../data/expense-categories.json)（アプリ側 `categories.json` と同期）。この文書は「どう選ぶか」を定める。

## 目的と方針

- 家計を **大分類 → 中分類 → 小分類** の3層で見られるようにする。粗すぎず細かすぎない粒度で、後から月次・カテゴリ別に振り返れる状態を保つ。
- **迷ったら浅い層で止めてよい**。L1（大分類）さえ正しければ記録の価値は保てる。確信が持てる層まで埋め、無理に深掘りしない。
- **一意に決める**。同じ品が複数カテゴリに等価で入らないよう、下の「割れやすいものの既定ルール」に従って所属を固定する。
- 食材台帳（`data/inventory` ほか）の更新と、家計簿への登録は**別の作業**。レシートが来たら両方行う。

## 3層の意味と必須度

| 層 | 名称 | 必須度 | 例 |
| --- | --- | --- | --- |
| L1 | 大分類（15固定） | **必須** | 食費 |
| L2 | 中分類 | 推奨 | 精肉・肉加工品 |
| L3 | 小分類 | 任意 | 鶏肉 |

- L1 は15種で固定: 食費 / 日用品 / 衣服・美容 / 交通 / 通信 / 住居 / 光熱 / 医療 / 教育 / 娯楽 / 交際費 / 保険 / 税・社会保険料 / 貯蓄・投資 / その他。
- L2 は L1 に属するものだけ、L3 は L2 に属するものだけが有効。`data/expense-categories.json` を唯一の正とする。
- アプリ側は無効値を親に落とす（無効な L1 → その他、L2 配下に無い L2 → 無視、L3 同様）。**ただし当てずっぽうで深い層を埋めない**。読めない層は空にする。

## 仕分け手順

1. **L1 を決める**（必須）。レシート全体の主用途を1つ選ぶ。店種ではなく「何の支出か」で見る（例: ドラッグストアでも、買ったのが食品なら食費、洗剤なら日用品）。
2. **L2 を決める**（推奨）。L1 配下の中分類から、確信が持てるものを選ぶ。判定できなければ空のままでよい。
3. **L3 を決める**（任意）。品目が明確なら付ける。`keywords` / `examples` に手がかりがある。曖昧なら空にする。
4. **割れやすいものは下表の既定ルールに従う**。レシート文言だけで判断できない要素（交際目的か・治療目的か等）は、既定の落とし所へ寄せる。
5. **レシート全体と明細を分ける**。レシート単位（receipt）は主用途のカテゴリ。明細単位（item）は行ごとの実態。1枚のレシートに食品と日用品と化粧品が混ざるなら、明細ごとに別カテゴリを付ける。明細のカテゴリを省くとレシート全体のカテゴリを継承する。

## 食費の中分類（売場・温度帯ベース）

食費の食材は、スーパーの売場と**温度帯（生鮮/チルド/常温/冷凍/即席）**を第一の手がかりに、次の中分類へ分ける（旧「食料品」は廃止し細分化した）。同じ食材でも状態で振り分ける（例: じゃがいも=生鮮なら青果、冷凍ポテトなら冷凍食品）。

- **精肉・肉加工品**: 生肉・ひき肉・内臓、ハム/ソーセージ/ベーコン、チルド調理用のミートボール/ハンバーグ等。
- **鮮魚・魚介**: 生鮮魚・刺身、貝/えび/いか/魚卵、ちくわ/かまぼこ等の練り物。
- **青果(野菜・果物)**: 生鮮の野菜・果物・きのこ、カット野菜/もやし/薬味。
- **卵・乳製品・豆腐納豆(日配)**: 牛乳/乳飲料/豆乳、ヨーグルト、チーズ/バター、卵、豆腐/納豆、漬物/キムチ、生麺/チルドデザート。**乳飲料はソフトドリンクでなくここ**。
- **主食(米・パン・麺・粉)**: 米/もち/パックごはん、食パン/菓子パン、乾麺/パスタ、粉、シリアル。**生麺=日配・冷凍麺=冷凍食品・即席麺=乾物即席**。
- **調味料・油・たれ・ソース**: 基礎調味料(塩砂糖醤油味噌酢みりん酒)、油、ソース/ドレッシング/マヨ/ケチャップ、だし/ルウ/コンソメ、スパイス。
- **乾物・缶詰・即席・レトルト**: 乾物/海藻、豆/ごま、缶詰/瓶詰、即席麺・カップ麺、即席スープ/カップスープ、レトルト/丼の素/ふりかけ。
- **冷凍食品**: 冷凍野菜/果物、冷凍肉/魚、冷凍調理品/おかず、冷凍麺/ごはん、アイス・冷菓。
- **飲料(ソフトドリンク)**: 水/お茶、コーヒー/紅茶、ジュース/炭酸/スポーツ、粉末・濃縮。**酒は酒類(家飲み)、乳飲料は日配**。
- **菓子・デザート**: スナック/米菓/乾き物、チョコ/ビスケット、和洋生菓子、製菓材料。
- 加えて、店内飲食=**外食**、持ち帰りの惣菜・弁当=**中食・惣菜**、家飲みの酒=**酒類(家飲み)**、判別不能=**その他食費**。

割れやすい食材の既定: 即席麺/カップ麺=乾物・缶詰・即席・レトルト、豆腐・納豆=日配、ドレッシング/たれ=調味料、できあがり惣菜の持ち帰り=中食・惣菜。状態が分かれば温度帯を優先（冷凍なら冷凍食品）。

## 旧「その他」だったものの移行先

以前は大分類が10種で、下記は「その他」に押し込まれていた。今は専用のL1がある。

| 服・下着・靴 | 衣服・美容 ▸ 衣類 / 靴・バッグ・小物 |

| ご祝儀・贈り物・会食 | 交際費 ▸ 冠婚葬祭 / 贈答・プレゼント / 会食・接待 |

| 税金・社会保険料・ふるさと納税 | 税・社会保険料 ▸ 税金 / 社会保険料 / ふるさと納税 |

## 登録方法（POST /api/receipts）

家計簿への登録は HTTP API で行う。

- エンドポイント: `POST http://127.0.0.1:8092/api/receipts`
- 認証: `Authorization: Bearer ${EXPENSES_WRITE_TOKEN}`（実値はサーバの `/opt/homebrew/var/lib/expenses-haya-homes/app/.env`。**実値はこのリポジトリに書かない**）
- カテゴリ: `category`（=L1, 必須）/ `categoryL2`（推奨）/ `categoryL3`（任意）。明細 `items[]` も同じキーを持てる。
- `source` は `kawasaki` を入れる。金額照合に必要な行（袋・値引き含む）は `items` に残す。
- 現在のカテゴリ定義は `GET http://127.0.0.1:8092/api/categories` でも確認できる（`data/expense-categories.json` と同じはず。ずれを感じたらこちらを正とする）。

```bash
curl -X POST http://127.0.0.1:8092/api/receipts \
  -H "Authorization: Bearer ${EXPENSES_WRITE_TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{
    "purchasedAt": "2026-06-28",
    "storeName": "ライフ 勝どき店",
    "category": "食費",
    "totalAmount": 1480, "paymentMethod": "クレジットカード", "source": "kawasaki",
    "items": [
      { "name": "鶏むね肉", "amount": 580, "category": "食費", "categoryL2": "精肉・肉加工品", "categoryL3": "鶏肉" },
      { "name": "ハンドクリーム", "amount": 900, "category": "衣服・美容", "categoryL2": "化粧品・スキンケア", "categoryL3": "スキンケア・フレグランス" }
    ]
  }'
```

成功すると `{"id": <レシートID>}` が返る。その id を食材台帳側の `purchase` イベント（`expense_registered`）に残し、二重登録を避ける。
